ساخت کاربر جدید در Django با CMD

برای ساخت یک کاربر جدید در پروژه Django، دو روش رایج وجود دارد.

روش اول: ساخت مدیر (Superuser)

اگر می‌خواهی یک کاربر مدیر (Superuser) بسازی، این دستور را در CMD یا ترمینال اجرا کن:

bash

python manage.py createsuperuser

پس از اجرای دستور، اطلاعات زیر از شما خواسته می‌شود:

* نام کاربری (Username)
* ایمیل (Email)
* رمز عبور (Password)
* تأیید دوباره رمز عبور

پس از تکمیل مراحل، این کاربر می‌تواند وارد پنل مدیریت Django شود که معمولاً در آدرس زیر در دسترس است:

http://localhost:<port>/admin

---

روش دوم: ساخت کاربر معمولی

اگر می‌خواهی یک کاربر معمولی ایجاد کنی، ابتدا وارد Django Shell شو:

bash

python manage.py shell

سپس کد زیر را اجرا کن:

python

from django.contrib.auth.models import User

user = User.objects.create_user(
username="ali",
password="12345678",
email="[ali@example.com](mailto:ali@example.com)"
)
user.save()

توضیح خط‌به‌خط

from django.contrib.auth.models import User
مدل داخلی User را از Django وارد می‌کند.

create_user(...)
یک حساب کاربری جدید ایجاد می‌کند.

user.save()
اطلاعات کاربر را در پایگاه داده ذخیره می‌کند.

نکات مهم

* از یک رمز عبور قوی استفاده کنید.
* اگر نام کاربری از قبل وجود داشته باشد، خطا دریافت خواهید کرد.
* قبل از اجرای دستورات، مطمئن شوید داخل پوشه پروژه قرار دارید.
* اگر پروژه از پایگاه داده استفاده می‌کند، ابتدا باید Migrationها را اجرا کرده باشید.

English

Creating a New User in Django Using CMD

There are two common ways to create a new user in a Django project.

Method 1: Create a Superuser

If you want to create an administrator (superuser), run the following command in CMD or your terminal:

bash

python manage.py createsuperuser

After running the command, you will be prompted to enter:

* Username
* Email address
* Password
* Password confirmation

Once completed, this user can log in to the Django Admin Panel, which is usually available at:

http://localhost:<port>/admin

---

Method 2: Create a Regular User

If you want to create a regular user, first open the Django shell:

bash

python manage.py shell

Then run the following code:

python

from django.contrib.auth.models import User

user = User.objects.create_user(
username="ali",
password="12345678",
email="[ali@example.com](mailto:ali@example.com)"
)
user.save()

Line-by-line Explanation

from django.contrib.auth.models import User
Imports Django's built-in User model.

create_user(...)
Creates a new user account.

user.save()
Saves the user to the database.

Important Notes

* Use a strong password.
* If the username already exists, Django will return an error.
* Make sure you are inside the project directory before running the commands.
* If your project uses a database, ensure that all migrations have already been applied.

العربية

إنشاء مستخدم جديد في Django باستخدام CMD

توجد طريقتان شائعتان لإنشاء مستخدم جديد في مشروع Django.

الطريقة الأولى: إنشاء مستخدم مدير (Superuser)

إذا كنت تريد إنشاء مستخدم مدير، شغّل الأمر التالي في CMD أو الطرفية:

bash

python manage.py createsuperuser

بعد تنفيذ الأمر، سيُطلب منك إدخال:

* اسم المستخدم
* البريد الإلكتروني
* كلمة المرور
* تأكيد كلمة المرور

بعد الانتهاء، يمكن لهذا المستخدم تسجيل الدخول إلى لوحة تحكم Django، والتي تكون متاحة عادةً على العنوان التالي:

http://localhost:<port>/admin

---

الطريقة الثانية: إنشاء مستخدم عادي

إذا كنت تريد إنشاء مستخدم عادي، افتح أولاً Django Shell:

bash

python manage.py shell

ثم نفّذ الكود التالي:

python

from django.contrib.auth.models import User

user = User.objects.create_user(
username="ali",
password="12345678",
email="[ali@example.com](mailto:ali@example.com)"
)
user.save()

شرح الأسطر

from django.contrib.auth.models import User
يستورد نموذج User المدمج في Django.

create_user(...)
ينشئ حساب مستخدم جديد.

user.save()
يحفظ بيانات المستخدم في قاعدة البيانات.

ملاحظات مهمة

* استخدم كلمة مرور قوية.
* إذا كان اسم المستخدم موجودًا مسبقًا، فستظهر رسالة خطأ.
* تأكد من أنك داخل مجلد المشروع قبل تنفيذ الأوامر.
* إذا كان المشروع يستخدم قاعدة بيانات، فتأكد من تنفيذ جميع عمليات الترحيل (Migrations) مسبقًا.
